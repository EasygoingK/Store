# Store
後端主要透過 DB First建立 Model，查詢、更新資料庫主要使用 Linq，前端使用 Razor語法，使用者權限使用表單驗證。

第一階段需求:
1.建立一個資料庫和資料表，用於存放 OrderList資料。
2.當 UserA送出Confirm請求時，更新資料庫 Status狀態，讓 UserA畫面看見目前訂單狀態為 To be shipped。
3.UserA 點擊產品名稱會以另一個頁面呈現詳細資料。

第二階段需求:
1.以表單驗證取得使用者 ID，再以使用者 ID區分有哪些訂單屬於自己的。
2.送出訂單後，將勾選的項目存入到另一個資料表，並關閉勾選功能，避免相同項目重複送出。
